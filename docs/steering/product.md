## プロダクト概要

カメラ（UVC/WEB カメラ）の映像に対して、リアルタイムで **物体検出・追跡**を行い、結果を GUI に表示するデスクトップアプリケーション。
GUI・撮像・推論/追跡を **multiprocessing** で別プロセスに分離することで、各処理を並列化し、UI のブロッキングを避けつつスループットを確保する。

## 想定ユーザー / ユースケース

- **想定ユーザー**: Python でリアルタイム映像処理・物体追跡を扱う開発者、マルチプロセス構成の参照実装を探している開発者。
- **主なユースケース**:
  - WEB カメラ映像から人物（既定では COCO class_id=0）を検出・追跡し、ID とクラスを一覧表示する。
  - 検出/追跡のフレームあたり処理時間と平均フレームレートをモニタリングする。
  - マルチプロセス + 共有メモリによるゼロコピー・フレーム転送の実装例として学習・流用する。

## 提供する主な価値（機能）

- 検出・追跡結果のオーバーレイ表示（カメラ画像にバウンディングボックス等を重畳）。
- 追跡 ID リスト表示（ID とクラス、最大10行。超過時は10行目を `...` 表示）。
- 性能表示（`xx.xx ms`, `xx.xx FPS`、フレームレートは直近最大100フレームの平均）。
- GUI ボタンによるプロセスの動的な開始/停止（起動時は「停止中」でカメラ非表示）。

## 非機能要求 / 設計上の狙い

- **リアルタイム性**: 遅い消費者（推論）がフレームを溜め込んで遅延が累積しないよう、フレーム読み出しポリシー（`fifo` / `latest` / `bounded_latest`）で最新フレームへ追従できる。
- **並列性 / 低コピーコスト**: プロセス間のフレーム転送は pickle を避け、共有メモリのリングバッファ（`SharedFramePool`）でゼロコピー転送する。
- **安全な停止**: `multiprocessing.Event` によりワーカーへ停止を通知し、リソースを解放してから終了する。
- **設定駆動**: カメラ・検出・追跡・GUI・ロギングの挙動を YAML 設定で外部化（実行時に `--config` で指定）。

## スコープ外 / 既知の制約

- カメラはローカルの UVC/WEB カメラ（既定インデックス 0）を前提。ネットワークカメラや録画ファイル入力は対象外。
- 検出モデルは YOLOX-S ONNX（`models/yolox_s.onnx`）を前提とした前後処理。
- ロギング出力先は `console` 固定（設定上のキーはあるが console 運用）。
